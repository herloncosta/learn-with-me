# AGENTS.md

## Project

Software engineering blog built with **Astro 5** + **MDX** + **Tailwind CSS v4**. Static site.

```
src/
├── content/blog/        # MDX posts (edit these to add articles)
├── content/config.ts    # Zod schema: title, description, pubDate, updatedDate?, draft
├── layouts/             # BaseLayout.astro (HTML shell)
├── components/          # Header, Footer, PostCard, PostList
├── pages/               # index.astro (homepage, 4 latest), blog/index.astro (infinite scroll), blog/[...slug].astro
├── styles/global.css    # Tailwind v4 @theme + OKLCH tokens + dark mode
└── scripts/             # page-transition.js (framer-motion page entrance)
```

## Commands

| Command | Action |
|---------|--------|
| `npm run dev` | Dev server (localhost:4321) |
| `npm run build` | Static build to `dist/` |
| `npm run preview` | Preview built site |

No test, lint, or typecheck scripts exist. No CI/CD.

## Framework quirks

- **Astro 5 content collections v3**: `glob()` loader + `render()` for `<Content />`. Not the legacy `content/` directory pattern.
- **Tailwind v4**: CSS-first — no `tailwind.config.mjs`. `@theme` block in `global.css`. `@custom-variant dark` instead of `dark:` modifier.
- **framer-motion**: page transitions (`page-transition.js`) and PostList infinite scroll use `animate()` from `framer-motion/dom`.
- **Colors**: OKLCH everywhere, CSS vars on `:root` / `.dark`, referenced by `@theme` for Tailwind utilities.
- **Dates**: locale `pt-BR`, timezone `UTC` — `.toLocaleDateString("pt-BR", { timeZone: "UTC", ... })`.
- **Dark mode**: `.dark` class on `<html>` via localStorage toggle in Header script (click) + BaseLayout IIFE (no-flash). Respects `prefers-color-scheme`.
- **Site URL**: `https://learn-with-me.dev` in `astro.config.mjs`.

## Content workflow

1. Create `.mdx` in `src/content/blog/`
2. Frontmatter: `title`, `description`, `pubDate`, optional `updatedDate`, `draft`
3. Restart dev server — route auto-generated by `blog/[...slug].astro`
